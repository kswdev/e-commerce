# Claude Code 기술 선택 가이드라인

## 판단 원칙

- 요청 규모가 작으면 직접 처리한다. 기술을 끼워 넣지 않는다.
- 새 기술 도입 전 "이게 없으면 못 하나?" 를 먼저 묻는다.
- 불확실하면 Plan Mode로 트레이드오프를 먼저 제시한다.

---

## 상황 → 기술 결정 매트릭스

| 상황 | 권장 기술 | 쓰지 않는 경우 |
|------|-----------|----------------|
| 처음 보는 코드베이스 영역 탐색 | Explore Agent (medium) | 익숙한 영역, 단순 파일 조회 |
| 아키텍처 결정 / 방향이 여러 갈래 | Plan Mode + Plan Agent | 구현 방법이 명확한 작업 |
| 독립적인 작업 2~3개 동시 처리 | 병렬 Agent (최대 3개) | 순서 의존성 있는 작업 |
| 새 페이지 추가 (반복 패턴) | `/new-page <PageName>` Skill | — |
| 반복 워크플로가 3회 이상 발생 | Skill로 추출 제안 | 일회성 작업 |
| 설계 결정을 세션 간 유지 | Memory (project 타입) | 코드로 표현 가능한 내용 |
| feature 브랜치 격리 개발 | Worktree | 단일 브랜치 작업 |
| 현재 작업 단계 추적 필요 | Task | 단순 1~2단계 작업 |
| 브라우저/DB/외부 API 직접 접근 | MCP *(보류)* | 백엔드 완성 전 |

---

## 현재 활성화된 자동화

| 종류 | 설정 | 동작 |
|------|------|------|
| Hook | PostToolUse (Write\|Edit) | tsc --noEmit + npm test 자동 실행 |
| Skill | `/new-page <PageName>` | 페이지 컴포넌트 + 라우트 자동 추가 |
| TDD | Red → Green → Refactor | 테스트 파일 먼저 작성 (Vitest + RTL) |

---

## 보류

- **MCP**: 백엔드 완성 후 재개. 그 전까지 `/data/*.json` 더미 데이터 사용.
